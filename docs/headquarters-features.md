# Особенности штабов

Документ описывает текущие штабы, их роль в балансе и ожидаемую игровую индивидуальность.

Источник актуальных данных: `tank-card-game/src/game/headquarters.ts`.

Важно: сейчас у штаба есть реальные числовые параметры (`hp`, `attack`, `range`, `fuelGeneration`, `level`, `defaultDeckId`) и текстовое описание. Большинство уникальных особенностей штабов пока являются дизайн-описанием, а не отдельной формализованной механикой в `engine.ts`. Если нужно сделать эти особенности рабочими правилами, лучше добавить структурированное поле вроде `headquartersAbilities`, а не парсить текст `description`.

## Текущая модель штаба

Каждый штаб содержит:

- `id` - стабильный идентификатор для колод, изображений, аватаров, прогресса и старта боя.
- `title` - короткое имя на карточке.
- `subtitle` - полное историческое/описательное имя.
- `type` - тип штаба или ветка развития.
- `nation` - нация: `ussr`, `germany`, `usa`, `uk`, `poland`, `france`.
- `hp` - здоровье штаба.
- `attack` - урон атаки штаба.
- `range` - дальность, сейчас обычно `99`, то есть почти всё поле.
- `fuelGeneration` - базовый прирост топлива.
- `level` - уровень штаба, влияет на вес колоды, подбор PVE-соперника и исследования.
- `defaultDeckId` - id стоковой колоды из `tank-card-game/src/game/initialState.ts`.
- `availableInMainMenu` - доступность в обычном меню; тестовые меню могут показывать штабы отдельно.

## Ветки штабов

Долгосрочно в игре четыре типа штабов:

- Танковые - давление, темп бронетехники, инициатива на поле.
- Мотопехотные - быстрый ввод войск, топливная эффективность, гибкая позиционная игра.
- Артиллерийские - усиленная огневая поддержка штаба или добивание повреждённых целей.
- Тыловые - экономика, ремонт, снабжение и затяжная игра.

Учебные штабы имеют 1 уровень. Основные штабы Германии, СССР и США сейчас введены на 4 уровне.

## Польские штабы кампании

Польские штабы сейчас используются в основном для первой кампании. Их колоды имеют id с суффиксом `_campaign`, а не обычные `_default`.

| ID | Название | Уровень | HP | Атака | Топливо | Роль |
|---|---|---:|---:|---:|---:|---|
| `polish_border_guard` | Пограничники | 2 | 13 | 1 | 3 | Ранняя оборона границы. |
| `polish_army_lodz` | Armia Łódź | 3 | 15 | 1 | 3 | Оборонительный армейский штаб. |
| `polish_army_prusy` | Armia Prusy | 3 | 16 | 1 | 4 | Резервная армия с более сильным давлением в долгом бою. |
| `polish_warsaw_defense` | Оборона Варшавы | 4 | 18 | 2 | 4 | Последний оборонительный рубеж с хорошей атакой и топливом. |

Польские штабы не стоит автоматически считать обычными быстрыми PVE/PVP штабами, пока не принято отдельное решение открыть кампанийный баланс для быстрых боёв.

## Правила колод и подбора соперника

Текущее состояние:

- Учебные стоковые колоды - 20 карт.
- Обычные неучебные стоковые колоды с суффиксом `_default` разворачиваются до 40 карт.
- Пользовательская колода - 40 карт плюс штаб.
- В PVE со стоковой колодой ИИ старается взять случайный обычный стоковый штаб того же уровня.
- В PVE с кастомной колодой ИИ ищет стокового соперника того же уровня с ближайшим весом колоды.
- Вес колоды теперь учитывает вес карт и интегральный вес штаба.
- Интегральный вес штаба учитывает уровень, здоровье, атаку, прирост топлива и дальность.

## Рекомендованная архитектура способностей штабов

Чтобы штабные особенности стали настоящими правилами, лучше добавить структурированную модель:

```ts
type HeadquartersAbility =
  | { type: "FIRST_TANK_MOVEMENT_BONUS"; amount: number }
  | { type: "FIRST_UNIT_COST_REDUCTION"; amount: number }
  | { type: "FIRST_HQ_ATTACK_DAMAGE_BONUS"; amount: number }
  | { type: "DAMAGED_TARGET_DAMAGE_BONUS"; amount: number }
  | { type: "LIGHT_UNIT_MOVEMENT_BONUS"; amount: number }
  | { type: "TURN_HEAL"; target: "hq" | "random_unit"; amount: number };
```

Затем добавить поле в `HeadquartersDefinition` и реализовать обработку в `engine.ts`. Так текст карточки, баланс и реальные правила не разъедутся.
