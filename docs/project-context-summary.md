# Суммаризация контекста проекта Panzershrek

Этот документ нужен как handoff для другой нейронки или разработчика, который продолжит работу без полного чата.

Рабочая папка: `C:\Prog\tank-card-game`.

Проект - WW2 карточная танковая игра на React + TypeScript + Vite с отдельным PVP-сервером. Пользователь быстро развивает игру итерациями: механики боя, PVP, кампании, исследования, декбилдер, UI, анимации, звук и экономику. Перед изменениями нужно читать актуальный код, потому что состояние проекта сильно изменилось.

## Общий дизайн игры

Игра - карточный тактический бой бронетехники.

Режимы:

- PVE - локальный бой против ИИ.
- PVP - онлайн-бой через сервер.
- Кампания - цепочка одиночных миссий.
- Исследования - открытие штабов и карт.
- Декбилдер - сборка пользовательских колод.

Ключевые элементы боя:

- У каждой стороны есть штаб.
- Юниты выводятся картами на поле.
- Есть линия поддержки/тыла - в интерфейсе называется `тыл`.
- Тыловые юниты делятся на артиллерию, транспорт и медиков.
- Обычный юнит за ход может один раз двигаться и один раз атаковать, если может.
- Стоимость хода как отдельная концепция была убрана.
- САУ/артиллерия должна держаться на дальних клетках и не идти в бессмысленную атаку на штаб.
- ПТ-САУ имеет особую механику: может ударить первой при обороне один раз за ход, с исключениями.
- Способность `Блиц` существует на картах: после выхода на поле юнит может сразу полноценно ходить и атаковать.
- Юниты поддержки не ходят и не атакуют напрямую.

## PVP

PVP был переведён к серверной авторитетности:

- Серверный таймер хода.
- Клиент только отображает время от сервера.
- Сервер сам завершает ход по таймеру.
- Реализованы сдача, выход из матча, отмена поиска, обработка disconnect.
- Сервер очищает таймеры и комнаты.
- Реализована приватизация BattleState: игрок не видит реальные карты руки и колоды противника.
- В PVP показывается, что противник выбирает карту.
- Экран ожидания PVP показывает выбранный штаб, флаг, карту мира, прицел, звук радара.
- Если соперник не найден за 50 секунд, запускается бой против ИИ с подходящим штабом.
- При fallback на ИИ не должно появляться сообщение о закрытом PVP-соединении.

Важное правило безопасности:

- Полный `BattleState` хранится только на сервере.
- Клиенты получают персональный `BattleStateView`.
- В PVP нельзя отправлять реальные `hand` и `deck` противника.

## ИИ и подбор соперника

ИИ дорабатывался много раз:

- Выбирает более опасные цели.
- Учитывает механику ПТ-САУ и не идёт в бессмысленные атаки.
- Бережёт САУ и держит их на дальних клетках.
- Не ведёт САУ к вражескому штабу, если нет хорошей причины.
- Может стрелять по штабу, если видит добивание дистанционными атаками.
- Защищает САУ танками/ПТ-САУ от быстрых атак.
- Умеет атаковать юниты на линии тыла артиллерией или штабом, если это выгодно.
- При кастомной колоде игрока бот подбирает колоду по весу.
- При стоковой колоде игрока бот играет случайным стоковым штабом того же уровня, когда возможно.

Вес колоды:

- Файл: `tank-card-game/src/game/deckWeight.ts`.
- Вес карты учитывает атаку, здоровье/броню, экономику, движение, дальность, способности, редкость, стоимость и класс.
- Вес штаба учитывает уровень, HP, атаку, прирост топлива и дальность.
- Итоговый вес колоды = вес штаба + вес карт.

## Штабы и колоды

Текущие учебные штабы:

- `training_unit` - СССР, сбалансированный учебный штаб.
- `trainingslager` - Германия, сильная атака, слабее экономика.
- `training_camp` - США, сильная экономика, слабее HP/атака.

Штабы 4 уровня:

- Германия: `first_panzer_division`, `german_motorized_division`, `german_artillery_division`, `german_rear_corps`.
- СССР: `soviet_tank_brigade`, `soviet_motor_rifle_division`, `soviet_guards_mortar_regiment`, `soviet_auto_battalion`.
- США: `usa_old_ironsides`, `usa_armored_infantry_regiment`, `usa_armored_artillery_battalion`, `usa_maintenance_battalion`.

Польские штабы кампании:

- `polish_border_guard`
- `polish_army_lodz`
- `polish_army_prusy`
- `polish_warsaw_defense`

Правила колод:

- Учебный штаб - максимум 20 карт в стоковой учебной колоде.
- Обычный неучебный стоковый штаб - 40 карт.
- Пользовательская колода - 40 карт плюс штаб.
- В учебные колоды можно брать карты разных наций.
- В неучебные колоды нужно брать карты своей нации.
- Для тестов в меню PVE/PVP показываются все обычные штабы 4 уровня со стоковыми колодами.

## Исследования и экономика

Реализуется дерево исследований по нациям и веткам штабов.

Валюты и опыт:

- Опыт конкретного штаба - основной опыт за бой, тратится на ветку этого штаба.
- Свободный опыт - примерно 7-10% общего опыта, можно тратить универсально.
- Железные траки - обычная валюта за бои, покупка карт и штабов.
- Золотые траки - премиум-валюта для будущих премиум-аккаунтов, платных кампаний и премиум-карт.

Экономические договорённости:

- Больше всего награды даёт победа в PVP через уничтожение полного HP противника.
- PVE даёт меньше награды.
- Стоимость карт и штабов должна расти экспоненциально.
- Если все карты штаба открыты, игра за этот штаб должна давать в основном свободный опыт.

В исследованиях уже есть:

- Сообщения, чего не хватает для исследования/покупки.
- Анимация покупки/исследования с крупной картой и рубашкой.
- Купленные копии отображаются стопкой карт.
- У открытого штаба должен показываться накопленный опыт штаба.
- Иконки опыта, железных и золотых траков используются в исследованиях и после боя.

## Декбилдер

Декбилдер сделан по образцу карточной игры:

- Верхняя линия - доступные карты.
- Нижняя линия - карты в колоде.
- Сначала выбирается штаб, он становится первой картой колоды.
- После выбора штаба список штабов заменяется доступными юнитами.
- Есть фильтры по типу юнита и нации.
- Можно добавлять/убирать карты кликом и drag-and-drop.
- Есть горизонтальный скролл колесом, мышью/тачем и стрелками.
- ПКМ открывает увеличенный вид карты.
- Есть поле имени колоды.
- После сохранения колода появляется в меню PVE и PVP.
- Последняя использованная колода должна показываться первой.
- В меню выбора боя каждая колода отображается как отдельная карточка штаба с подписью.
- В декбилдере показывается вес колоды и вес карты при наведении.

Визуальные пожелания пользователя:

- Не добавлять лишние рамки и полоски.
- Свечение при drag-and-drop должно быть мягким золотым, под картами.
- Стрелки скролла не должны перекрывать карты.

## UI, визуал и ассеты

Общий стиль: тёмный военный интерфейс, тактические карты, металл, латунь, приглушённое свечение.

Шрифты:

- `Rajdhani` - латинские display-заголовки и цифровые показатели.
- `Inter` - весь кириллический UI и обычный текст.
- CSS-переменные находятся в `tank-card-game/src/index.css`.

Кнопки:

- Используется изображение `tank-card-game/src/assets/button.png`.
- Серые варианты - пауза, выход, отмена.
- Базовый цвет - важные действия вроде конца хода/готово.

Фоны:

- В PVE/PVP фон боя выбирается случайно.
- В миссиях позже должны задаваться конкретные фоны.
- Фоны должны храниться отдельно и подхватываться автоматически.
- Если фон обрезается, нужно править `object-fit`/позиционирование, а не руками резать каждую картинку без нужды.

Флаги:

- Нации: СССР, Третий рейх/Германия, США, Британия, Польша, Франция.
- Флаги используются на карточках, штабах и виджетах профиля.
- Пользователь уже вручную подгонял размеры флагов, менять осторожно.

Аватары штабов:

- Загружаются по id штаба из отдельной папки.
- В бою аватар игрока снизу слева.
- Аватар противника справа сверху.
- Нижний край аватара должен иметь мягкое появление/исчезновение, без ощущения обрезки.

## Боевые анимации и звук

Анимации:

- Стрелка движения из `arrow.png`, зелёная для своих, красная для врага.
- Для лёгких танков движение на две клетки анимируется поэтапно: стрелка, движение, стрелка, движение.
- При уроне иконка щита увеличивается, раскалывается, из неё вылетает число урона.
- Юнит дёргается по направлению удара.
- Уничтожение использует `burnt-card`: карта сгорает, съёживается и улетает к колоде.
- Выбранный юнит/штаб подсвечивается тонким золотым свечением.
- Возможные клетки хода подсвечиваются мягким зелёным пульсом.
- Доступные цели атаки подсвечиваются красным в стиле выбранного юнита.
- Тыловые ячейки квадратные, без золотых рамок.

Звук:

- В меню играет `main`.
- В бою плавно начинается `battle`.
- Выстрелы - случайный звук из `sounds/cannon_shot`.
- Выдача карты, выход карты на поле и движение карты - случайный звук из `sounds/card_distrib`.
- Начало хода - `steel_imp`.
- Уничтожение - `paper_burning`.
- Экран поиска PVP - случайный звук радара из `sounds/radar`.

## Экран результата

Экран победы/поражения переделан на основе файлов из `game/result_screen`.

Требования:

- Победа зелёным, поражение красным.
- Кнопка должна вести в меню, не начинать новый бой.
- Вкладка `Сводка` показывает опыт и траки.
- Вкладка `Трофеи` показывает уничтоженные и потерянные юниты.
- Классы в таблице: лёгкие, средние, тяжёлые, ПТ-САУ, САУ, тыловые войска.
- Числа опыта и железных траков на баннере должны анимироваться как счётчик топлива, но чуть дольше.

## Профиль игрока

В меню, кроме боя/декбилдера/дерева развития, отображается аккаунт:

- Аватар любимого штаба.
- Ник игрока.
- Подпись базовый/премиум аккаунт.
- Фон на основе флага нации любимого штаба.
- Любимый штаб по умолчанию - советский учебный штаб.
- Количество сыгранных матчей по штабам должно фиксироваться, чтобы корректно выбирать любимый/используемый штаб.

В профиле:

- Общая статистика.
- Статистика побед/поражений.
- Список штабов.
- Для каждого штаба - нация, бои, победы, поражения, опыт.

## Рабочий процесс

В рабочем дереве часто есть много незакоммиченных изменений. Нельзя откатывать чужие изменения без явной просьбы.

Обычный порядок:

1. Найти актуальный код через `rg`.
2. Прочитать конкретные файлы через `Get-Content`.
3. Внести правки через `apply_patch`.
4. Если менялся код - запустить `npm.cmd run lint`.
5. Если менялся код - запустить `npm.cmd run build`.
6. В ответе перечислить изменённые файлы и результат проверки.
