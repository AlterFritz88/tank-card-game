# Интеграция магазина с ЮKassa

## Цель

Магазин клиента показывает пакеты золотых траков и покупку премиум аккаунта. Покупка премиума за уже имеющиеся золотые траки выполняется через профильный сервер. Покупка золотых траков за реальные деньги должна выполняться только через серверный платежный сценарий.

## Товары

Золотые траки:

- 100 золотых траков
- 500 золотых траков
- 1500 золотых траков

Премиум аккаунт:

- 1 день: 99 золотых траков
- 5 дней: 470 золотых траков
- 21 день: 1500 золотых траков
- 50 дней: 4199 золотых траков

## Рекомендуемый flow ЮKassa

1. Клиент выбирает пакет золотых траков.
2. Клиент получает каталог с ценами через `GET /api/shop/catalog`.
3. Клиент отправляет на сервер `POST /api/shop/gold-payment` с `playerId` и `productId`.
4. Сервер проверяет товар и создает платеж в ЮKassa.
5. Сервер сохраняет локальную запись платежа: `paymentId`, `playerId`, `productId`, `amount`, `goldTracks`, `status`.
6. Сервер возвращает клиенту `confirmationUrl`.
7. Клиент открывает страницу оплаты.
8. ЮKassa отправляет webhook на `POST /api/payments/yookassa/webhook`.
9. Сервер заново запрашивает платеж в API ЮKassa, проверяет статус и idempotency.
10. Только после успешного webhook сервер начисляет золотые траки профилю игрока.
11. Клиент при следующей синхронизации получает обновленный баланс.

## Что нельзя делать

- Нельзя начислять золотые траки по сообщению клиента после возврата со страницы оплаты.
- Нельзя доверять цене или количеству золотых траков, которые прислал клиент.
- Нельзя хранить секрет ЮKassa в клиентском коде или Vite env.
- Нельзя повторно начислять золото по одному `paymentId`.

## Переменные окружения сервера

- `YOOKASSA_SHOP_ID`
- `YOOKASSA_SECRET_KEY`
- `YOOKASSA_RETURN_URL`
- `YOOKASSA_GOLD_100_RUB`, опционально, по умолчанию `99`
- `YOOKASSA_GOLD_500_RUB`, опционально, по умолчанию `449`
- `YOOKASSA_GOLD_1500_RUB`, опционально, по умолчанию `1190`
- `YOOKASSA_FIRST_PLAYER_PACK_RUB`, опционально, по умолчанию `199`
- `PLAYER_PAYMENT_DB_PATH`, например `/data/player-payments.json`

Цены в `YOOKASSA_GOLD_*_RUB` используются и для создания платежа, и для отображения цены в клиентском магазине через `GET /api/shop/catalog`. Если env-переменная не задана, сервер использует цену по умолчанию из каталога.

Опционально:

- `YOOKASSA_API_BASE`, если нужен тестовый/прокси endpoint вместо `https://api.yookassa.ru/v3`.

## Серверные сущности

Минимальная запись платежа:

```ts
type GoldPaymentRecord = {
  paymentId: string;
  playerId: string;
  productId: "gold-100" | "gold-500" | "gold-1500";
  goldTracks: number;
  amountRub: number;
  status: "pending" | "succeeded" | "cancelled";
  createdAt: number;
  updatedAt: number;
  creditedAt: number | null;
};
```

На файловом хранилище Amvera можно начать с отдельного JSON-файла в `/data`, но перед публичным релизом лучше перейти на БД с уникальным индексом по `paymentId`.

## Текущий статус реализации

Сервер уже умеет:

- создавать платеж ЮKassa для пакетов `gold-100`, `gold-500`, `gold-1500`;
- отдавать клиенту каталог пакетов золота с рублевыми ценами;
- сохранять платежи в `PLAYER_PAYMENT_DB_PATH`;
- принимать webhook ЮKassa;
- перепроверять платеж через API ЮKassa;
- начислять золотые траки только один раз по ключу `payment:{paymentId}`.
